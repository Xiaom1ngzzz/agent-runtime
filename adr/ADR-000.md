# ADR-000: Architecture Decision Record 模板

- **状态**: Accepted
- **日期**: 2026-07-09
- **决策者**: —

## Context

本仓库采用 ADR 来记录关键的架构决策。ADR-000 定义模板,其后的每条 ADR 从此处 fork。

## Decision

采用如下结构编写 ADR:

```
# ADR-XXX: 标题

- 状态: Proposed | Accepted | Superseded by ADR-YYY | Deprecated
- 日期: YYYY-MM-DD
- 决策者:

## Context
决策背景、驱动力、约束。

## Decision
最终选择及其边界。

## Consequences
正向、负向、需要接受的取舍。

## Alternatives
考虑过但未选择的选项,以及未选择的原因。

## References
相关 issue / PR / 章节 / 论文链接。
```

## Consequences

- 决策历史可追溯。
- 新成员可以按时间线理解演进。

## Alternatives

- 无 ADR,仅用提交信息记录 —— 上下文易丢失。
- 使用 wiki 记录 —— 与代码分离,容易过期。

## References

- Michael Nygard, *Documenting Architecture Decisions* (2011)
